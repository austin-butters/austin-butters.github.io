An analogy to describe JavaScript and its relationship to HTML and CSS.

If we compare creating a website to building a house, HTML acts as the structure. It includes the foundation, the framing, and things like windows and doors. CSS is everything aesthetic, including the paint, curtains, the interior design, and the trimmings like skirting boards and windowsills. Finally, JavaScript is the plumbing and electricity. It flows through the house’s walls unseen, and adds interactivity and functionality. It makes the lights turn on when you flick a switch, makes the water flow, etc.

To summarize, HTML says what’s there, CSS says how it looks, and JavaScript says how it behaves.

Describe what the DOM is and an example of how you might interact with it.

DOM is short for “Data Object Model”. The DOM is a model that represents HTML documents as nested objects. It can be visualized as a tree, where the root is the html element, branches are elements like header, body, and the leaves are elements like <p> tags, images, etcetera. By nesting elements in this way, each element has a position in the DOM that we can use to “do stuff”.

HTML might seem like an unnecessarily complicated language. It divides documents into elements with classes, IDs, and many different properties. Why don’t we just upload PDFs or images of what we want the site to look like? There’s a few reasons for this, but one major reason becomes obvious when we introduce JavaScript. HTML is an abstraction of the way a document looks, which makes it easy for JS to interact with. By dividing documents into elements and organizing them with classes, IDs, <div>s, etc, we make it easy for JS to interact with the HTML document’s content.

We can access and manipulate specific elements and their properties in many ways. For example, let’s assume we have this HTML element: 

 <h1 class=“redTitle” id=“greeting”>Hello!<h1>

We can interact with this in a number of ways. First, we have to “grab a hold of” the element, so that the JavaScript knows what element it’s manipulating. We can do this in a few ways:

.getElementbyId()

Grabs an element with the id greeting. It returns a single element, as every HTML document should only have one element with any ID.

.querySelector()

Returns an element based on a CSS selector. This can be:

  1. A tag: h1
  2. A class: .redTitle
  3. An ID: #greeting

The querySelector method will only return the first matching element in the HTML document. To return all elements, we would use the method .querySelectorAll()

.getElementsByClassName()

Returns all elements with a matching class name. These elements will be returned in a type of JavaScript object known as a NodeList.

.GetElementsByTagName()

Returns all elements with a matching tag (in this example, all h1 elements) in a Node List

Explain the difference between accessing data from arrays and objects.

There are many types of data in JavaScript. Values are usually stored as numbers, strings, or booleans. We can also use arrays and objects to store groups of values.

An array stores information in a list, separated by commas. For example:

let names = [“Alice”, “Bob”, “Charlie”] // This is an array.

An object stores data in key-value pairs:


        let names = {
          name1: "Alice",
          name2: "Bob",
          name3: "Charlie"
        }; // this is an object

Arrays and objects are both infinitely nestable. This means a value in an array or object can be another array or object itself. This is very useful, although deep nesting can cause performance issues in more complicated situations. Here’s an example of a nested object:

In this example, we have an object called “people”. Within that object we have nested objects called “person1”, “person2”, “person3” for individual people. Objects allow us to store much more complicated information than arrays, as each value has a key to identify it. For example, we can see Alice’s height, Bob’s age, and Charlie’s cafe order. Notice that we have an “order” array nested within the object as one of the values.
